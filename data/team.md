## Процессы

### Спринты
- Длительность: 2 недели
- Планирование: первый день спринта (понедельник)
- Демо: последний день спринта (пятница)
- Ретроспектива: после демо
- Daily standup: каждый день в 10:00, формат: что сделал / что планирую / блокеры

### Код-ревью
- Каждый PR должен получить минимум 1 approve от ревьюера
- Ревьюер назначается по зоне ответственности:
  - Backend-код → Иван
  - Frontend-код → Мария
  - DevOps/CI → Алексей
  - Тесты → Елена
- Время на ревью: максимум 24 часа с момента создания PR
- Автор не может approve свой собственный PR

### Деплой
- Staging: автоматический деплой при мерже в main (после прохождения CI)
- Production: ручной деплой после тестирования на staging
- Rollback: через git revert + повторный деплой
- Ответственный за деплой: Алексей Козлов (devops)
- Hotfix-процесс: ветка hotfix/* от main, ускоренное ревью, деплой в тот же день

## Definition of Done

Задача считается выполненной, когда:
1. Код написан и соответствует стилю проекта (Kotlin coding conventions)
2. Написаны юнит-тесты с покрытием ≥ 80% нового кода
3. PR создан, прошёл код-ревью и получил approve
4. CI пайплайн прошёл без ошибок (сборка + тесты + линтинг)
5. Документация обновлена (если затронут публичный API)
6. Задача протестирована на staging
7. Нет открытых блокирующих багов

## Приоритизация задач

### Правила приоритизации
- **Critical**: Блокирует работу команды или пользователей. Немедленное исправление. Пример: падение production, потеря данных.
- **High**: Важная функциональность или дедлайн в ближайшие 3 дня. Берётся в работу в текущем спринте обязательно.
- **Medium**: Плановая задача спринта. Берётся по порядку после high.
- **Low**: Nice-to-have, можно перенести на следующий спринт.

### Дополнительные факторы приоритета
- Задача, которая **блокирует другие задачи**, повышается в приоритете
- Задача с ближайшим **дедлайном** имеет приоритет над задачей с далёким дедлайном при одинаковом уровне
- **Загруженность исполнителя**: если у человека уже 3+ задачи in_progress, новые задачи ему не назначаются
- При конфликте приоритетов решение принимает вся команда на daily standup

### Статусы задач
- **todo**: Задача создана, ожидает начала работы
- **in_progress**: Задача в работе
- **review**: Задача на код-ревью (PR создан)
- **done**: Задача выполнена (соответствует Definition of Done)
